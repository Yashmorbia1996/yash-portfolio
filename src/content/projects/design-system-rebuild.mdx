## Overview

This project involved a ground-up rebuild of the internal design system that had grown organically over 3 years and accumulated significant technical debt. The system was used across 8 product teams with different codebases and tech stacks.

## Problem

The legacy component library had no consistent API, poor documentation, and zero test coverage. Teams had forked components to meet local needs, creating a fragmented UI and slow feature development.

## Approach

I worked closely with design and 4 engineering leads to define a token-based system with clear component APIs. We adopted Radix UI primitives for accessibility and built a Storybook instance for documentation and visual regression testing.

## Outcome

- 70% reduction in reported UI inconsistencies
- 2× faster design-to-dev handoff (measured over 3 sprints)
- 100% of new UI work built on the new system within 6 months
- Open-sourced the token generator as a standalone tool
