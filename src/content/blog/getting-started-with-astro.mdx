Astro 5 is one of the most exciting framework releases in recent memory. If you haven't tried it yet, here's what you're missing.

## What Makes Astro Different

Unlike React or Next.js frameworks that ship JavaScript by default, Astro renders to static HTML at build time. JavaScript is opt-in — you add it only where interactivity is needed.

This "islands architecture" means:

- **Faster page loads** — less JS to parse and execute
- **Better SEO** — fully rendered HTML, no hydration delay
- **Simpler mental model** — pages are templates, not apps

## Content Collections

Content collections are Astro's killer feature for portfolio sites and blogs. You define a schema with Zod, then write content in MDX files. The framework validates your frontmatter at build time, giving you full type safety.

```typescript
const blog = defineCollection({
  loader: glob({ pattern: '**/*.mdx', base: './src/content/blog' }),
  schema: z.object({
    title: z.string(),
    date: z.coerce.date(),
    tags: z.array(z.string()),
  }),
});
```

## Astro + React

Astro integrates with React, Vue, Svelte, and more. Use framework components only where you need interactivity, and let Astro handle everything else as static HTML.

```astro
---
import MyReactComponent from '../components/MyComponent.tsx';
---

<!-- This renders as static HTML -->
<h1>Hello, world</h1>

<!-- This ships React JS only for this component -->
<MyReactComponent client:load />
```

## Getting Started

Run `pnpm create astro@latest` and follow the prompts. Or, if you're using this starter template, you're already set up — just update `src/config/site.ts` with your details and start writing content.
